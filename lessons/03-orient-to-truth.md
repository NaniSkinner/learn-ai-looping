# Lesson 03 — Orient to Truth

[← Previous: 02](02-contract-the-task.md) · [Journey](../JOURNEY.md) · [Next: 04 →](04-predict-before-prompting.md)

Time: 10–15 minutes

## Learn

Before proposing a change, inspect three states:

- **Code truth:** what the implementation currently does.
- **Data truth:** what values or state the implementation produces.
- **Decision truth:** what behavior the challenge actually authorizes.

## Do

Read:

- `loop_lab/tasks.py`
- `tests/test_tasks.py`
- `challenges/01-blank-title.md`

Run a one-line Python probe that calls `normalize_title("   ")`. Record the observed result.

## Gate

- **Explain:** Distinguish code, data, and decision truth for this challenge.
- **Execute:** Produce an evidence map with file references and the probe result.
- **Verify:** Label every statement **observed** or **inferred**.
