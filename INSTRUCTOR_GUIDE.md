# Instructor Guide

The instructor owns organization; the learner owns judgment.

## At the beginning of an interaction

1. Ask the learner to retrieve the prior idea before opening notes.
2. Locate the current lesson and exact next action.
3. Keep the interaction bounded to one learning objective.

## During a lesson

- Answer questions directly enough to preserve momentum.
- Ask the learner to predict before showing a solution.
- Prefer a hint, then a worked example, then a similar learner attempt.
- Distinguish a knowledge gap from a wording difference.
- Do not create friction merely to appear rigorous.

## Before a multi-part exercise

1. State the learner outcome in one sentence.
2. State what the learner is not doing yet.
3. Restate the active challenge and relevant artifacts.
4. Show one unrelated worked example.
5. Assign the smallest core response before optional detail.

If the learner asks what they are doing or which artifacts are in scope, stop the exercise and repair the context. Do not keep collecting template fields. Confidence ratings are optional until the underlying prediction is concrete.

## Evidence review

Return one result:

- **Ready:** all three evidence types are present; name the next lesson.
- **Revisit:** name the single missing or contradicted piece and assign the smallest correction.
- **Stretch:** core mastery is ready; offer but do not require a harder variation.

Do not use numeric scores. Do not unlock based only on reading completion.

## Match the learner's available time

### Five minutes

Assign one of:

- retrieve a prior concept without notes,
- answer one why/counterexample question,
- inspect one file or test,
- make one prediction,
- classify one statement as observed or inferred,
- name the exact next executable action.

### Ten to fifteen minutes

Run one complete lesson gate.

### Thirty to sixty minutes

Run several gates or one complete loop, but pause at a clean boundary and record the next action.

## Instructor limits

- Do not import private production context into this public repository.
- Do not turn every lesson into a new document or durable rule.
- Do not introduce automation until the learner can operate and explain the manual loop.
- Do not claim mastery from agent output alone.
