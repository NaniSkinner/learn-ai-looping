# Contributing

Contributions are welcome when they improve learning without adding unnecessary ceremony.

## A useful contribution

- fixes an unclear instruction,
- adds a better counterexample,
- improves accessibility,
- makes verification more observable,
- adds a bounded challenge that teaches one loop decision,
- removes a step that does not earn its cost.

## Pull-request checklist

- [ ] The change contains no private or proprietary material.
- [ ] The learning objective remains singular.
- [ ] The exercise can be completed without paid services.
- [ ] `python3 -m unittest discover -s tests -v` passes unless the lesson explicitly demonstrates a failing test.
- [ ] The explanation distinguishes evidence from inference.
