# Lesson 01 — Name the Loop

[← Previous: 00](00-start-here.md) · [Journey](../JOURNEY.md) · [Next: 02 →](02-contract-the-task.md)

Time: 10–15 minutes

## Learn

The foundational loop is:

> Frame → Orient → Predict → Plan → Execute → Verify → Explain → Preserve

The loop exists because a fluent output is not evidence that the task is understood or complete.

## Do

In `learner/progress.md`, explain each stage in one sentence without copying the wording below the arrow.

Then identify which stage is missing from each behavior:

1. “The agent changed the file and said tests pass, so I accepted it.”
2. “I asked for a solution before writing what I expected to be true.”
3. “I start every new session by retelling the same repository rules.”

## Gate

- **Explain:** Define all eight stages in your own words.
- **Execute:** Diagnose the three missing stages.
- **Verify:** Give one example of an output that looks good but would fail the loop.
